# リリース前アナウンス内容

- 固定 IP アドレスの追加（NAT Gateway の切り替えのため）
- 新しい URL での Webhook の追加
    - [kintoneのヘルプページ](https://jp.cybozu.help/k/ja/user/app_settings/set_webhook/webhook.html)によると、10個まで追加できるとのこと。10個使い切ってるお客様は、リリース直後にURLを変えてもらうくらいかなぁ。。
- その他、リリース前の対応は不要なものの、予告だけするやつ：
    - リリースする機能の紹介（パスワードポリシーの設定機能）
    - プラグインのバージョンアップがリリース後に必要なこと

# リリース後アナウンス内容

- 古い IP の削除のお願い
- 古い URL の Webhook の削除
- プラグインのバージョンアップのお願い
    - [こちら](https://novelworks.slack.com/archives/GK8SYKPND/p1666253743500959)でお伝えしたとおり、ユーザーの画面にアップデートを促す文言を表示する対応はするが、メールでもアナウンスするのが丁寧と思う。
